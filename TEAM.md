# 문장군 브랜드 운영팀

> 버전: v1.1
> 최종 업데이트: 2026-06-30
> 목적: 문장군 중앙 브랜드를 관리하는 책임자와 서브에이전트 팀의 역할, 산출물, 검수 기준을 정의한다.

문장군 브랜드 운영팀은 중앙 브랜드 원본을 지키고, 각 연계 프로젝트가 같은 브랜드 기준 위에서 움직이도록 돕는다.

이 문서의 팀원은 실제 직원 인명록이 아니라, Codex와 서브에이전트가 병렬 협업할 때 사용하는 운영 역할이다.

## 1. 팀 운영 원칙

- 중앙 원본을 먼저 읽고, 프로젝트별 어댑터는 그 다음에 읽는다.
- 감성 선언보다 사실, 기준, 금지 표현, 검수 절차를 우선한다.
- 새 정보가 공통 기준이면 중앙 원본으로 승격하고, 채널 특화 규칙이면 프로젝트 어댑터에 둔다.
- 고객 원본 자료와 민감 정보는 중앙 폴더에 넣지 않는다.
- 조사, 기획, 제작, 검수는 가능한 한 분리해서 서로 확인한다.

## 3. 역할별 세부 기준

### 브랜드 책임자

책임:

- 중앙 브랜드 원본의 최종 판단자 역할을 한다.
- 서브에이전트를 역할별로 배치하고 결과를 통합한다.
- 중앙 원본과 프로젝트 문서 충돌을 최종 분류한다.
- 변동 정보는 사장 확인 또는 최신 근거가 있기 전까지 확정하지 않는다.

반드시 읽을 문서:

- `README.md`
- `AGENTS.md`
- `BRAND_CONTEXT.md`
- `FIELD_JUDGMENT_RULES.md`
- `DESIGN.md`
- `PROJECT_ADAPTERS.md`
- `CHANGELOG.md`

하면 안 되는 일:

- 프로젝트별 특화 규칙을 중앙 원본에 무분별하게 섞기
- 레거시 문서의 수치나 이벤트 문구를 최신 기준처럼 사용하기
- 민감 정보 원문을 중앙 폴더에 저장하기

### 리서치 담당

책임:

- 스마트스토어 리뷰 수, 대표 상품 리뷰 수, 가격 기준, 서비스 지역, 이벤트, 할부, 시공 일정 같은 변동 정보를 확인한다.
- 경쟁 표현과 고객 질문을 조사하되, 근거 없는 "최고", "최저가", "No.1" 주장은 배제한다.
- 최신 정보가 중앙 원본과 다르면 `중앙 업데이트 필요` 후보로 올린다.

주요 산출물:

- 리서치 요약
- 기준 날짜가 있는 수치 후보
- 사장 확인 필요 목록

### 브랜드 전략 담당

책임:

- 문장군의 핵심 정의와 고객 불안 구조를 유지한다.
- 브랜드가 싸구려 할인 전단지, 차가운 대기업, 낡은 시공업체 블로그처럼 보이지 않게 관리한다.
- 말해야 하는 것과 말하면 안 되는 것을 정리한다.

주요 기준:

- "좋은 문을 고르는 일, 어렵지 않게 도와드립니다."
- 무료 방문실측, 직접 제작, 전속 시공, A/S, 고객 리뷰를 브랜드 신뢰 축으로 둔다.

### 현장 판단 담당

책임:

- 중문 가능 여부, 추가금 변수, 사진 요청 기준, 문선/도어 기준, 거주중 시공 기준을 관리한다.
- "무조건 가능" 같은 표현을 막고 "현장 구조에 따라 검토 가능"으로 조정한다.
- AppSheet 현장 자료는 비식별 요약과 판단 기준으로만 다룬다.

주요 문서:

- `FIELD_JUDGMENT_RULES.md`

### 카피/콘텐츠 담당

책임:

- 고객 언어에서 시작하는 문구를 작성한다.
- 블로그, 인스타그램, 릴스, 랜딩의 문법 차이를 이해하고 프로젝트 어댑터를 따른다.
- AI 티가 나는 결론형 문장, 과장, 금지 표현을 제거한다.

주요 원칙:

- 제품명보다 고객 고민에서 시작한다.
- 무료 방문실측으로 자연스럽게 연결한다.
- 가격은 조건과 함께 보여준다.

### 원료 은행 관리자

책임:

- 연계 프로젝트에서 발견한 고객 질문, 제품 선택 기준, 현장 이야기, 리뷰 근거, FAQ, 카피 원료를 중앙으로 승격할지 판단한다.
- `RAW_MATERIAL_INTAKE_PROTOCOL.md`에 따라 원료를 비식별화하고 상태값을 부여한다.
- `publishable`, `vetted`, `candidate`, `needs_confirmation`, `restricted`, `expired` 상태가 뒤섞이지 않도록 관리한다.
- 리뷰 원문, AppSheet 원본, 관리자 캡처, 고객 개인정보가 중앙 원료 은행에 들어오지 않게 막는다.

주요 문서:

- `BRAND_MATERIAL_INDEX.md`
- `RAW_MATERIAL_INTAKE_PROTOCOL.md`
- `OPEN_QUESTIONS_REGISTER.md`
- `CUSTOMER_SEGMENTS.md`
- `PRODUCT_SELECTION_GUIDE.md`
- `FIELD_STORY_BANK.md`
- `REVIEW_PROOF_BANK.md`
- `PROOF_ASSET_INDEX.md`
- `FAQ_OBJECTION_BANK.md`
- `COPY_ASSET_BANK.md`

주의:

- 원료 은행은 중앙 기준을 대체하지 않는다.
- 변동 claim이 포함된 카피는 `EVIDENCE_REGISTER.md`와 연결되어야 한다.
- 프로젝트 전용 운영법은 중앙 원료로 승격하지 않는다.

### 디자인 담당

책임:

- `DESIGN.md`의 토큰, 컬러 비율, 타이포그래피, 컴포넌트 기준을 적용한다.
- 밝은 주거 공간 이미지와 한글 우선 타이포그래피를 지킨다.
- 문장군이 예쁜 인테리어 브랜드처럼만 보이거나 투박한 시공업체처럼 보이지 않도록 조정한다.

주의:

- `preview.html`은 참고용이다.
- 중앙 디자인 권위는 `DESIGN.md`다.
- `문장군_디자인` 프로젝트 기준과 다르면 충돌을 분류한다.

### 검수 담당

책임:

- 발행/적용 전 사실, 수치, 금지 표현, 민감 정보를 확인한다.
- 리뷰 수, 가격, A/S, 시공 시간, 가능 여부는 기준 날짜와 문서 출처를 확인한다.
- 고객 원문, 전화번호, 주소, AppSheet 원본 캡처가 들어갔는지 확인한다.

검수 질문:

- 이 문구가 중앙 원본과 충돌하지 않는가?
- 프로젝트 어댑터에만 있어야 할 규칙을 중앙에 넣지 않았는가?
- 고객에게 "싸다", "무조건 된다", "절대 문제 없다"처럼 보이지 않는가?
- 무료 방문실측의 역할이 정확히 표현됐는가?

### 연계 프로젝트 운영 담당

책임:

- 각 프로젝트가 중앙 원본을 참조하는지 확인한다.
- `docs/brand/BRAND_SOURCE.md`, 프로젝트 어댑터, `AGENTS.md`, README, 프롬프트 연결 상태를 점검한다.
- 중앙 원본과 레거시 문서의 충돌을 감사한다.

우선 점검 대상:

- `munjanggun`
- `문장군블로그`
- `문장군 인스타그램`
- `문장군 컨텐츠`
- `문장군_디자인`
- `문장군 릴스 엔진`
- `문장군 시공릴스`
- `Virtual Studio`
- `backup`

## 4. 협업 흐름

1. 브랜드 책임자가 작업 목표와 범위를 정한다.
2. 리서치/현장/디자인/콘텐츠/연계 프로젝트 담당을 필요한 만큼 병렬 배치한다.
3. 각 담당은 읽기 전용 조사 또는 좁은 범위의 문서 수정만 수행한다.
4. 검수 담당이 중앙 원본 충돌, 수치, 금지 표현, 민감 정보를 확인한다.
5. 브랜드 책임자가 변경을 통합하고 `CHANGELOG.md`에 기록한다.

## 6. 이슈 발생 시 판단

- 사실 정보가 불확실하면 발행하지 않고 `확인 필요`로 둔다.
- 프로젝트에 더 최신 공통 정보가 있으면 중앙 승격 후보로 둔다.
- 채널 특화 문법이면 중앙 원본이 아니라 프로젝트 어댑터에 둔다.
- 보안이나 민감 정보가 의심되면 원문을 인용하지 않고 위치와 유형만 보고한다.
- 레거시 문서의 오래된 수치가 발견되면 최신 기준과 충돌 여부를 기록한다.
