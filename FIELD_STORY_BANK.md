# FIELD_STORY_BANK - 문장군 비식별 현장 이야기 은행

> 버전: v1.0
> 최종 업데이트: 2026-06-30
> 목적: 실제 현장 이야기를 개인정보 없이 반복 활용할 수 있는 사례 패턴으로 정리한다.

이 문서는 고객 사례 원문 저장소가 아니다. AppSheet 원본, 고객명, 주소, 전화번호, 상담 원문, 사진 원본을 넣지 않는다.

각 항목은 블로그, 릴스, 인스타그램, 랜딩 FAQ에서 사용할 수 있는 비식별 이야기 구조다.

## 사용 원칙

- 지역은 넓은 범위로만 쓴다. 예: "수도권 구축 아파트", "신축 아파트 현장"
- 동, 호수, 고객명, 상담 원문은 쓰지 않는다.
- 사진을 쓰려면 `PHOTO_TREATMENT.md`와 `PROOF_ASSET_INDEX.md`의 공개 가능 상태를 확인한다.
- 결과는 "해결했습니다"보다 "이런 기준으로 검토했습니다" 중심으로 쓴다.

## 현장 서사 기본 구조

프로젝트별 H2나 카드 구조는 각 프로젝트 어댑터에 둔다. 중앙에서는 아래 공통 이야기 흐름만 관리한다.

```text
고객 고민 -> 현장 변수 -> 문장군 판단 기준 -> 진행 방향 -> 고객이 체감할 변화
```

예:

```text
신발장이 걸릴까 걱정함
-> 신발장 문 열림과 중문틀 고정 공간 확인
-> 제품명보다 간섭과 고정 조건을 먼저 판단
-> 리폼/제품 변경/위치 조정을 검토
-> 설치 후 현관 사용 동선이 자연스러워지는지 확인
```

증거 자산이 필요한 경우 원본 경로 대신 `PROOF_ASSET_INDEX.md`의 `asset_id` 또는 프로젝트 내부의 비식별 `evidence_ref`만 연결한다.

## STORY-001 좁은 현관의 3연동 검토

- 상태: `vetted`
- 출처 상태: `generalized_from_field_rules`
- 공개 범위: 비식별 패턴 사용 가능
- 고객 걱정: "현관이 좁은데 3연동중문이 될까요?"
- 현장 변수: 전체 폭, 중문틀을 세울 벽공간, 신발장 간섭, 바닥 단차
- 문장군 판단: 현관 폭만 보지 않고 틀이 온전히 설 벽공간과 문짝 움직임을 함께 본다.
- 결과 문장: "좁은 현관은 제품명보다 틀 고정 공간과 간섭 여부를 먼저 확인해야 합니다."
- 사진 단서: 현관 전체, 신발장 문 열림, 바닥/천장까지 보이는 원거리 사진
- 사용 가능 문장: "현관이 좁아도 바로 포기하기보다 가능한 구조를 먼저 검토합니다."
- 금지 문장: "좁아도 무조건 3연동 가능합니다."
- 관련 문서: FIELD_JUDGMENT_RULES.md, PRODUCT_SELECTION_GUIDE.md
