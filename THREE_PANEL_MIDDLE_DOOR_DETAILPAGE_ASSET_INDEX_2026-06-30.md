## 4. 자동화용 manifest 기준

`asset-manifest.json`은 132개 파일 전체를 아래 필드로 관리한다.

자동화는 파일 시스템 순회 순서나 JSON 배열 순서만 믿지 않고, `assemblyOrder`, `folderRole`, `collection`, `sequence`를 기준으로 조립한다. 현재 manifest 배열도 이 순서로 정렬되어 있지만, 소비 규칙은 명시 필드를 우선한다.

| 필드 | 뜻 |
| --- | --- |
| `assetId` | 자동화/문서에서 참조할 안정 ID |
| `path` | 자동화 호환용 경로 alias. `relativePath`와 같은 값 |
| `sequence` | 같은 폴더 안 진열 순서 |
| `relativePath` | 중앙 저장소 기준 파일 경로 |
| `folderRole` | 공통 도입, 이벤트/공지, 컬러, 유리, 컬렉션 본문, 썸네일 역할 |
| `collection` | `basic`, `modern`, `classic`, `art` |
| `fileType` | `jpg`, `png`, `gif` |
| `dimensions` | `{ width, height }` 형식의 자동화 호환용 규격 객체 |
| `width`, `height` | 이미지 규격 |
| `frameCount` | GIF 프레임 수 |
| `bytes` | 파일 크기 |
| `sha256` | 동일 파일/중복 관계 검사용 해시 |
| `duplicateGroup` | 컬렉션 간 공유 썸네일 관계 |
| `usageStatus` | 후보/확인 필요 상태 |
| `claimRisk` | 이미지 안에 포함된 가격/이벤트/조건 claim 위험 |
| `privacyStatus` | 공개 전 개인정보 검수 상태 |

## 5. 주요 발견

1. 상세 본문 이미지는 대부분 860px 폭의 세로형 이미지다.
2. GIF는 23개이며, 컬러/컬렉션 상세에서 움직임으로 옵션 차이를 보여주는 역할이 크다.
3. `컬러` 섹션은 36.0MB 수준으로 가장 무겁고, 10MB급 GIF가 포함된다.
4. `컬렉션/클래식`은 `001~017` 순서가 모두 채워져 있다. 2026-06-30 사용자가 005 이미지를 새로 추가하고 기존 005를 006으로 변경한 상태를 manifest v1.1에 반영했다.
5. 썸네일 폴더에는 컬렉션 간 동일 SHA256 이미지가 있다. 이는 삭제 대상이 아니라 공유 옵션/공통 썸네일 관계로 취급한다.
6. `이벤트 및 공지` 섹션에는 이벤트, 월 납입, 리뷰/혜택성 문구가 포함되어 보이므로 최신 조건 확인 전 외부 발행에 사용하지 않는다.
