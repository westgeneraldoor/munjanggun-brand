# EVIDENCE_REGISTER - 문장군 변동 정보 근거표

> 버전: v1.1
> 최종 업데이트: 2026-06-30
> 목적: 리뷰 수, 가격, A/S, 일정처럼 시간이 지나면 바뀔 수 있는 customer-facing claim의 기준일, 출처, 확인자, 재확인 주기를 관리한다.

이 문서는 `BRAND_CONTEXT.md`, `FIELD_JUDGMENT_RULES.md`, `DESIGN.md`, 각 프로젝트 어댑터에서 사용하는 변동 claim의 근거표다.

원칙:

- 새 콘텐츠, 랜딩, 광고, 카드뉴스, 릴스, 블로그에 변동 claim을 쓰기 전 이 표를 확인한다.
- 표의 `상태`가 `candidate`, `needs_confirmation`, `restricted`, `expired`이면 외부 발행 전 최신 근거와 사용 범위를 확인한다.
- 최신 근거가 확인되면 이 문서와 관련 중앙 문서, `CHANGELOG.md`를 함께 업데이트한다.
- 확인되지 않은 숫자는 단정 표현으로 쓰지 않는다.
- 원료 은행 문서에서 변동 claim이 포함된 문구를 쓰려면 이 문서의 상태와 사용 기준을 먼저 확인한다.

## Claim Status

| 상태 | 뜻 | 사용 기준 |
| --- | --- | --- |
| `publishable` | 최신 근거와 사용 범위가 확인됨 | 조건과 범위를 함께 외부 발행 가능 |
| `vetted` | 방향은 확인됐지만 상황별 예외 설명 필요 | 일반 안내로 사용 가능 |
| `candidate` | 근거 후보가 있으나 최신성 또는 범위 확인 필요 | 내부 기획만 가능. 외부 발행 전 재확인 |
| `needs_confirmation` | 사장 확인, 최신 캡처, 정책 확인 필요 | 외부 발행 금지 |
| `restricted` | 원본이나 민감 정보 포함 가능성 있음 | 비식별 요약 또는 공개용 재제작 전 사용 금지 |
| `expired` | 기준일 만료 또는 운영 변경 가능성 큼 | 최신 근거 재등록 전 사용 금지 |

## Safe Claim Copy Rules

| 상황 | 허용 방향 | 금지 방향 |
| --- | --- | --- |
| 리뷰 수 | 상품/전체/예약 리뷰 범위를 구분하고 기준일을 함께 관리 | 대표 상품 리뷰를 전체 브랜드 리뷰처럼 표현 |
| 가격/월 납입 | 조건, 범위, 실측 후 정확 견적 안내와 함께 사용 | 최저가, 확정가, 추가금 없음처럼 보이게 표현 |
| A/S | 기간, 범위, 예외 확인 후 FAQ나 신뢰 섹션에서 설명 | 평생 보증, 무조건 무상 |
| 일정 | 일반 안내로 쓰고 운영 상황에 따라 달라질 수 있음을 설명 | 특정 날짜나 기간을 무조건 약속 |
| 시공 가능 여부 | 현장 구조에 따라 검토한다고 표현 | 무조건 가능 |
| 이벤트/할부 | 기준일, 기간, 조건 확인 후 통제된 화면에서 사용 | 상시 혜택처럼 표현 |

## Update Rule

변동 claim을 수정할 때는 아래 파일을 함께 검토한다.

```text
EVIDENCE_REGISTER.md
BRAND_CONTEXT.md
FIELD_JUDGMENT_RULES.md
PROMPTS.md
CHANGELOG.md
```

프로젝트별 최신 운영 수치가 발견되면 바로 중앙 원본에 덮어쓰지 않는다. 먼저 `중앙 업데이트 필요`로 분류하고, 출처와 기준일을 확인한 뒤 중앙 문서에 반영한다.
