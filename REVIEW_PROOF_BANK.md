# REVIEW_PROOF_BANK - 리뷰와 후기 근거 은행

> 버전: v1.0
> 최종 업데이트: 2026-06-30
> 목적: 리뷰 원문이 아니라, 리뷰가 어떤 고객 불안을 해소하는지와 어떤 표현까지 안전한지 정리한다.

이 문서는 리뷰 원문 저장소가 아니다.

중앙 폴더에는 리뷰 작성자명, 닉네임, 원문 전문, 관리자 캡처, 고객 개인정보를 넣지 않는다. 실제 리뷰 문구를 인용하려면 원본 캡처, 사용 권한, 최신성, 개인정보 제거 여부를 별도 확인해야 한다.

## 1. 리뷰 근거 상태

| 상태 | 뜻 |
| --- | --- |
| `theme_only` | 원문 없이 리뷰에서 자주 확인되는 불안 해소 주제 |
| `needs_source` | 실제 캡처나 링크 확인 후 사용 가능 |
| `publishable_quote` | 공개 사용 권한과 개인정보 검수가 완료된 짧은 인용 |
| `restricted` | 내부 확인용. 외부 발행 금지 |

현재 중앙 은행에는 `publishable_quote` 상태의 실제 리뷰 인용을 저장하지 않는다.

## 2. 리뷰가 증명할 수 있는 것

| 불안 | 리뷰 근거 유형 | 안전 표현 | 상태 |
| --- | --- | --- | --- |
| 상담이 어렵지 않을까 | 친절한 상담, 설명이 쉬웠다는 후기 | "상담 과정이 어렵지 않았다는 후기가 신뢰를 더합니다." | `theme_only` |
| 견적이 불안하다 | 실측 후 조건을 설명받았다는 후기 | "실측 상담에서 조건을 확인하고 결정했다는 후기 흐름을 활용합니다." | `needs_source` |
| 마감이 괜찮을까 | 마감이 깔끔했다는 후기 | "마감 만족 후기는 제품보다 시공 결과를 보여주는 근거입니다." | `theme_only` |
| 일정이 걱정된다 | 일정 안내와 방문 경험 후기 | "일정 안내 만족은 확정 약속이 아니라 운영 신뢰 근거로만 씁니다." | `needs_source` |
| 사용감이 궁금하다 | 부드럽게 열리고 닫힌다는 후기 | "사용감 후기는 After 사진과 함께 쓰면 설득력이 높습니다." | `theme_only` |
| A/S가 걱정된다 | 시공 후 응대 경험 후기 | "A/S 관련 후기는 보증 조건과 함께 확인한 뒤 사용합니다." | `needs_source` |
| 디자인 선택이 어렵다 | 집 분위기와 잘 어울렸다는 후기 | "색상과 유리 선택 만족은 디자인 선택 불안을 줄이는 근거입니다." | `theme_only` |

## 3. 리뷰 수 표현

리뷰 수는 `EVIDENCE_REGISTER.md`를 우선한다.

안전한 방향:

- "고객 리뷰로 검증된 브랜드"
- "실제 후기로 확인하는 도어·중문 시공"
- "대표 상품 리뷰와 전체 상품 리뷰는 범위를 구분해 사용합니다."

주의:

- 대표 상품 단일 리뷰 수를 전체 브랜드 리뷰처럼 쓰지 않는다.
- 과거 `리뷰 15,000개+` 표현을 전체 브랜드 리뷰처럼 쓰지 않는다.
- 최신 캡처 없이 숫자를 확대하지 않는다.

## 4. 리뷰 인용 사용 절차

1. 원본 리뷰 위치를 확인한다.
2. 작성자명, 닉네임, 주문정보, 지역, 사진 속 개인정보를 제거한다.
3. 인용은 짧게만 사용한다.
4. 리뷰 수나 날짜가 변동 claim이면 `EVIDENCE_REGISTER.md` 기준일을 확인한다.
5. `PROOF_ASSET_INDEX.md`에 증거 자산 상태를 남긴다.

## 5. 금지

- 리뷰 원문 전문 저장
- 작성자 닉네임 저장
- 관리자 화면 캡처 저장
- 특정 고객을 식별할 수 있는 후기 재구성
- "고객만족 100%" 같은 단정
- "모두가 만족" 같은 전면화
