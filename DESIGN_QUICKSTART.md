# DESIGN_QUICKSTART - 문장군 디자인 실무 요약

> 버전: v1.1
> 최종 업데이트: 2026-06-29
> 기준 문서: DESIGN.md v3.0

이 문서는 `DESIGN.md`를 매번 전부 읽기 어려운 실무자와 에이전트를 위한 1장 요약입니다. 최종 권위는 항상 `DESIGN.md`에 있습니다.

## 한 줄 원칙

문장군 디자인은 예쁜 도어를 파는 디자인이 아니라, 고객이 좋은 문을 덜 불안하게 고르도록 돕는 디자인입니다.

## 브랜드 인상

```text
첫인상: 우리 집이 예뻐질 것 같다.
중간 설득: 내 집 구조도 봐줄 것 같다.
신뢰 근거: 직접 제작하고 전속 시공한다.
전환 이유: 무료 방문실측으로 정확히 알 수 있다.
마지막 안심: A/S와 리뷰 근거가 있다.
```

## 핵심 문장

공식 브랜드 핵심 문장:

```text
좋은 문을 고르는 일,
어렵지 않게 도와드립니다.
```

디자인/히어로 변형 문장:

```text
좋은 문을 고르는 일,
따뜻하게 정확하게
```

변형 문장은 랜딩 히어로, 캠페인, 섹션 대표 카피에서 사용할 수 있지만 공식 핵심 문장을 대체하지 않습니다.

## 디자인 3축

- Warm Craft: 흙, 나무, 크림, 테라코타에서 오는 장인적 온기.
- Guided Choice: 고객이 혼자 고르지 않도록 고민별·분위기별 선택지를 좁혀주는 구조.
- Verified Work: 무료 방문실측, 직접 제작, 전속 시공, 리뷰, A/S로 신뢰를 증명하는 구조.

## 색상 사용

| 역할 | 토큰 | 기준 |
| --- | --- | --- |
| 깊이/장인성 | `--mg-cocoa-oak` | 제목, 신뢰 표면, 직접 제작·전속 시공 근거 |
| 상담 행동 | `--mg-clay-terracotta` | CTA, 상담 예약, 전환 구간 |
| 기본 배경 | `--mg-cashmere-cream` | 첫 화면, 브랜드 스토리, 따뜻한 주거감 |
| 보조 표면 | `--mg-almond-soft` | 선택 가이드, 상담 흐름 |
| 검증 신뢰 | `--mg-verified-navy` | 리뷰 수치, 운영, 전산, A/S 근거 |

한 화면 비율은 크림/화이트 60%, 아몬드 18%, 오크 12%, 테라코타 7%, 네이비 3%를 기준으로 합니다.

## 서체 사용

- 히어로, 섹션 제목, 브랜드 문장: `Noto Serif KR`
- 본문, FAQ, 폼, 버튼, 탭: `Pretendard` 또는 `Noto Sans KR`
- 숫자, 영문 보조 표기, 라벨: `Manrope` 또는 `Inter`
- 전체 UI를 세리프로 만들지 않습니다.
- 모바일 한글 줄바꿈을 반드시 확인합니다.

## 이미지 사용

- 첫 화면은 밝은 주거 공간 속 완성된 중문/도어 이미지가 우선입니다.
- 공장, 작업장, 장비, 시공 중 현장, Before/After는 근거 섹션에서 사용합니다.
- 고객명, 전화번호, 상세 주소, 차량 번호, AppSheet 원본 화면은 노출하지 않습니다.
- 사진 보정 기준은 `PHOTO_TREATMENT.md`를 따릅니다.
- 이미지 구도, 최소 해상도, 텍스트 안전영역은 `DESIGN.md`와 `tokens/brand.tokens.json`의 media 토큰을 확인합니다.

## 컴포넌트 사용

- 첫 화면 신뢰 배지는 3개만 사용합니다.
  - 무료 방문실측
  - 직접 제작·전속 시공
  - 고객 리뷰 검증
- 가격은 숨기지 말되 조건과 함께 설명합니다.
- 리뷰 수, 가격, A/S, 일정, 시공 시간처럼 변동 가능한 claim은 `EVIDENCE_REGISTER.md`를 확인합니다.
- `Proof Panel`은 "우리가 최고"가 아니라 확인 가능한 근거를 보여주는 영역입니다.
- 상담 CTA는 테라코타를 쓰되 할인 전단지처럼 보이면 실패입니다.

## 구현 토큰

권위 관계는 다음 순서입니다.

```text
DESIGN.md front matter
→ tokens/brand.tokens.json
→ tokens/brand.css
→ preview.html 및 각 프로젝트 구현
```

명칭 변환 규칙:

- `DESIGN.md`는 원본 문서 가독성을 위해 kebab-case와 `rounded`를 사용합니다.
- `tokens/brand.tokens.json`은 구현 도구 호환을 위해 camelCase와 `radius`를 사용합니다.
- `tokens/brand.css`는 `--mg-*` CSS 변수명을 사용합니다.
- 이름은 달라도 값과 역할은 같아야 합니다.

토큰을 바꿀 때는 `DESIGN.md`를 먼저 수정하고, `tokens/brand.tokens.json`, `tokens/brand.css`, `preview.html`, `CHANGELOG.md`를 함께 갱신합니다.

## 출시 전 12문항 체크

1. 첫 화면이 밝은 주거 공간 또는 따뜻한 크림 배경으로 시작하는가?
2. 네이비가 첫인상 주색이 되지 않았는가?
3. 테라코타가 할인/특가 느낌으로 과하게 쓰이지 않았는가?
4. 첫 화면 신뢰 배지가 3개를 넘지 않는가?
5. 고객 고민에서 선택 흐름이 시작되는가?
6. 가격 안내가 조건, 실측, 추가금 설명과 함께 있고, 변동 claim은 `EVIDENCE_REGISTER.md` 기준을 확인했는가?
7. 공장/현장 사진이 첫 화면이 아니라 근거 섹션에 있는가?
8. `Noto Serif KR`이 큰 제목 중심으로만 쓰였는가?
9. 본문과 폼은 산세리프로 읽기 쉽게 구성됐는가?
10. 모바일에서 "무료 방문실측", "직접 제작", "전속 시공" 같은 단어가 어색하게 잘리지 않는가?
11. 금지 표현이 쓰이지 않았는가?
12. 고객 개인정보와 AppSheet 원본 정보가 노출되지 않았는가?
